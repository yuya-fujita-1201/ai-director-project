## 問題点

### 1. ハッシュタグの一貫性がない
- シリーズタグ（#AI監督プロジェクト）が記事①③に**ない**
- タグの粒度がバラバラ（#AI vs #ClaudeOpus vs #Claude）
- 記事②はタグが3つしかない（5つが推奨）
- 検索ボリュームの大きいタグ（#プログラミング #ノーコード #Flutter）が使われていない

### 2. タイトルが検索キーワードを意識していない
- ストーリー性は高いが、Google検索で引っかかる具体的キーワードが不足
- 記事①は47文字で長すぎ（推奨28〜32文字、最大40文字程度）
- 「AI アプリ開発」「コード書かない アプリ」「App Store 申請 AI」など検索されそうなフレーズが埋もれている

### 3. 記事間の内部リンクが不十分
- マガジンリンクは記事④のみ（他の記事にはない可能性）
- 記事同士の相互リンクがない → Googleのクロール効率が下がる

### 4. 見出し（H2/H3）にキーワードが不足
- 見出しが物語調で、検索キーワードを含んでいないケースがある

## 改善案

### A. ハッシュタグ統一戦略

**全記事共通タグ（3つ）:**
- `#AI監督プロジェクト` — シリーズ識別・ブランディング
- `#AIでアプリ開発` — メインキーワード（ミドルワード）
- `#個人開発` — ビッグワード（note内で人気）

**記事別タグ（残り2つ）:**

| 記事 | 追加タグ1 | 追加タグ2 |
|------|----------|----------|
| ① | `#AI` | `#iOSアプリ` |
| ② | `#プログラミング初心者` | `#Flutter` |
| ②.5 | `#Cowork` | `#Claude` |
| ③ | `#Claude` | `#iOSアプリ` |
| ④ | `#AppStore` | `#Cowork` |

### B. 記事本文への内部リンク追加

各記事の**冒頭**または**末尾**に以下を追加:

```
📖 このシリーズの記事一覧
① 企画編（無料）→ リンク
② 環境構築編（無料）→ リンク
②.5 ツール解説編（無料）→ リンク
③ 実装編（無料）→ リンク
④ App Store申請編（¥500）→ リンク
⑤ 結果報告編（¥980）→ 公開後追加

👉 マガジンでまとめ読み → リンク
```

**効果:**
- Google: 内部リンク構造の強化 → クロール効率UP + 各記事の評価連動
- note内: シリーズ回遊率UP → PV・スキ増加 → note推しアルゴリズムで露出増

### C. ハッシュタグ変更の優先順位

**まずやるべき（効果大・工数小）:**
1. 全記事のハッシュタグを統一戦略に合わせて更新
2. 記事①③にシリーズタグ `#AI監督プロジェクト` を追加

**次にやるべき（効果中・工数中）:**
3. 各記事末尾にシリーズ一覧リンクを追加
4. 各記事末尾にマガジンリンクを追加

**余裕があれば（効果中・工数大）:**
5. 記事本文の見出しにキーワードを自然に組み込む
6. 記事冒頭に検索意図に答える1-2文を追加

## 実施方法

noteの公開済み記事を更新するフロー:
1. 記事ページ → 「...」→「編集」→ エディタ表示
2. 本文の末尾にシリーズリンクを追加（HTMLペースト）
3. 「公開に進む」→ 公開設定ページ
4. ハッシュタグを変更
5. 「記事の追加」でマガジン追加を確認
6. 有料記事の場合: 「有料エリア設定」→ 有料ラインを確認
7. 「更新する」

※ 注意: 更新時にマガジン追加が外れることがあるので、必ず確認する
