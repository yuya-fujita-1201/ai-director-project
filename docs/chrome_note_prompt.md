## 「AI、すごいらしい」で止まっていませんか？

2026年2月、Anthropic社が最新AI「Claude Opus 4.6」をリリースした。

「業界最高水準の知能」「エージェント型コーディングで圧倒的性能」――ニュースやXではそんな言葉が飛び交っている。正直、何を言っているのかさっぱりわからない。

でも、ひとつだけ引っかかったことがあった。

最近のClaude、チャットで日本語を打ち込むだけで「ファイルを整理して」「ブラウザでこの作業やって」「コードを書いて」みたいなことが本当にできるらしい。しかも全部Macの上で。

自分はIT企業で働いてはいるけど、プログラミングは学生の頃にHTMLをちょっとかじったくらい。Flutterって何？ Firebaseって何？ 正直そのレベルだ。

でも思ったのだ。

AIへのチャット指示だけで、iOSアプリを作って、App Storeに出して、お金を稼ぐところまでいけたら……めちゃくちゃ面白くないか？

というわけで、確かめてみることにした。

## まず、Claudeに相談してみた

使ったのは claude.ai。ブラウザでアクセスして、チャット画面に日本語で話しかけるだけ。LINEやChatGPTを使ったことがある人なら、まったく同じ感覚で使える。

で、最初に送ったメッセージがこれ。

> 自分：
> LINEスタンプ、iOSアプリ作成、Webページ作成、Note、X運用。
> これらのうちマネタイズするならどれが良いかな？
> まずは月数万円程度の収入を目標にしようと思うんだけど。
> AIの活用には多少詳しいので、それを活かせるものがいい。

要するに「副業で月数万円稼ぎたいんだけど、何がいい？」という、ざっくりした相談だ。

返ってきた回答が驚くほど具体的だった。各選択肢を「月数万円に届く現実性」で比較したうえで、「Note × X × iOSアプリ」の3層構造がベストという戦略を出してきた。

・短期（1〜2ヶ月）：Note有料記事で月数万円を作る
・並行：Xで日々の発信 → Noteへ誘導
・中長期：iOSアプリでサブスク収益を積み上げる

……え、3分でここまで出てくるの？　自分が1人で考えてたら、たぶん3日は悩んでいたと思う。

## ここで「逆転の発想」を思いついた

ここまでは順調だ。でも、ここでひとつ問題が出てきた。

「じゃあアプリ作ろう」となったとき、特に作りたいアプリがないのだ。

これ、たぶん同じことを感じている人は多いと思う。「何か作りたい」という気持ちはあるのに、いざ企画を考えると手が止まる。市場調査とか競合分析とか、真面目にやればやるほど「これじゃない」感が積み上がって、結局何も始まらない。

……で、ふと思った。

企画で悩むくらいなら、企画もAIに丸投げすればいいのでは？

普通は「自分で企画→AIで実装→記事にする」という流れだろう。でも、逆にしてしまう。

「AIに企画を出させる → その通りに実行する → その過程を全部記事にする」

こうすれば、企画で悩む時間はゼロ。しかも1つのプロジェクトから自動的に3〜5本の記事が生まれる。成功しても失敗しても記事になる。「AIの言う通りにやったら本当に稼げた」も面白いし、「AIの言う通りにやったのに大失敗した」も面白い。

これを「AI監督プロジェクト」と名付けることにした。

自分は「監督（AI）の指示に従う俳優」みたいなものだ。

## Claude Opus 4.6の「4つの顔」を知った

ここで少し脱線するけど、今回のプロジェクトを進めるうえで知っておくべきことがある。

「Claude」って1種類のツールだと思っていたのだが、調べてみたら2026年現在、Macでは4つの使い方ができるらしい。これがけっこう衝撃だった。しかも全部、日本語のチャットで動く。

① Claude.ai ― 「参謀」
ブラウザで開くチャットAI。今あなたが読んでいるこの記事の企画も、ここで相談しながら作った。「何を作ればいい？」「どう売ればいい？」みたいな、ざっくりした相談に乗ってくれる頭脳担当。ChatGPTを使ったことがある人なら、ほぼ同じ感覚。

② Cowork ― 「事務員」
Claude Desktopというアプリに入っている機能で、これがすごい。Mac上のフォルダを指定すると、その中のファイルを読んだり整理したりしてくれる。「このフォルダの中身を整理して」で本当に整理してくれる。触ってみたとき「え、これ秘書じゃん」と思った。

③ Claude in Chrome ― 「手足」
Chromeブラウザの拡張機能。Webサイト上の操作をAIが代わりにやってくれる。ページの要約はもちろん、「このフォームに入力して」「このボタンを押して」までやる。今回はNoteの投稿やApp Storeでの作業に使う予定。

④ Claude Code ― 「エンジニア」
ターミナル（Macの黒い画面）で動くコーディング専門のAI。「○○なアプリを作って」と言うと、コードを書いて、テストして、エラーを直してくれる。正直、ターミナルを開くだけでビビるレベルの自分だが、AIが全部やってくれるので問題ない……はず。

参謀・事務員・手足・エンジニア。この4人のAIチームを、自分は日本語のチャットで指揮するだけだ。すごい時代になったものだ。

## AI監督に「売れるアプリを企画して」と頼んでみた

さて、本題に戻ろう。

Claude Opus 4.6に、自分の状況を伝えたうえで「月数万円を目指せるアプリを企画して」とお願いした。

伝えたスペックはこんな感じ。

・Macを持っている
・iPhoneアプリを出したい（Apple Developer Programは登録済み）
・開発に使える時間は週10〜20時間
・プログラミングは実質未経験（全部AIに任せたい）

すると、Claudeは3つの企画を出してきた。それぞれ「なぜこの企画か」「技術的な見積もり」「収益モデル」まで添えて。こっちはざっくり頼んだだけなのに。

企画A：ビフォーアフター記録アプリ
→ 写真をスライダーで比較できるアプリ。筋トレやDIYの記録に。

企画B：今日だけTODOアプリ
→ 毎日リセットされるTODO。タスクは最大5つまで。

企画C：AI英語フレーズカメラ「SnapEnglish」
→ モノを撮影するとAIが英語フレーズを生成してくれるアプリ。

## AI監督の判断：「企画Cで行きます」

3つ出されて「どれにする？」と聞かれたので、「AI監督が決めてください」と丸投げした。

だってこのプロジェクト、全部AIに任せるのがルールだから。自分で選んだら意味がない。

Claudeの回答：

> 企画C「SnapEnglish」で行きます。

理由がまた面白かった。

「AIが企画したAI活用アプリを、AIで実装する」という入れ子構造が記事として最も面白い。英語学習市場は日本のApp Storeで常にトップカテゴリだから市場性がある。API連携の過程は技術記事としてのネタが豊富で、1プロジェクトから3〜5本の記事が書ける――と。

……待って。「コンテンツとしての面白さ」まで考慮して企画を選んでいる。自分が「記事にしたい」と言ったのを汲んで、記事映えする企画を推してきたのだ。ちょっと怖いくらい賢い。

## AI監督から「1週間のスケジュール」が提示された

Claudeはさらに、1週間のロードマップまで出してきた。毎日やることが決まっている。

【Day 1】 環境構築 + カメラ機能
【Day 2】 AI連携（写真を送ったら英語フレーズが返ってくる）

……1週間？

プログラミング未経験の自分が、チャットだけで、7日間でApp Storeにアプリを出す？

無謀すぎる。でも、AI監督がそう言うのだから従うしかない。無謀だから記事になるのだ。

## このプロジェクトのルール

「AI監督プロジェクト」として、自分なりにルールを決めた。

ルール①：AIの判断には基本的に従う
「その企画は嫌だ」とか「ここはこうしたい」とか言わない。監督の指示には従う。

ルール②：自分がやるのは「日本語でチャット」だけ
Claude.ai、Cowork、Claude in Chrome、Claude Code。4つのツールを使うけど、やることは全部「日本語を打ち込む」だけ。コードは一行たりとも自分で書かない。

ルール③：過程を全て記録する
うまくいったことも、盛大にコケたことも隠さない。チャットのスクショも全部出す。

ルール④：収益は全公開する
DL数、課金率、売上。Note記事の閲覧数や収益も。全部オープンにする。

## ちなみに、この記事もAIに書いてもらいました

ここまで読んでくれた方にひとつ白状がある。

この記事自体が、AI監督プロジェクトの成果物第1号だ。

Claude Opus 4.6に「今日のやり取りの経緯をNote記事にして」と頼んだら、下書きが出てきた。それを読んで、「ここはちょっと違うな」「この表現は自分っぽくないな」というところだけ直して、Noteに投稿した。

自分がやったことは：
1. Claudeとチャットする（「副業どうしよう」的な相談）
2. 「この内容を記事にして」と頼む
3. 出てきた下書きを読んで、自分の言葉に近づける
4. Noteに投稿する

チャット開始から記事公開まで、約2時間。

「え、それでいいの？」と思うかもしれないけど、いいのだ。これが2026年のコンテンツ制作だと思っている。もちろん、最終的に読んで確認して「これは自分の記事として出せるな」と判断しているのは自分自身だ。でも、ゼロから書いたわけではない。

なお、Note自体もほぼ初めて使ったのだけど、そのへんの話は番外編「Noteの始め方ガイド」で詳しく書く予定だ。アカウント作成から投稿方法まで、AIに聞きながらやった過程を全部載せる。
