## 1. Cowork（Claude Desktop）の実践的な使い方

### やれたこと
- **プロジェクトフォルダを指定して丸ごと管理**: docs/, articles/, logs/ などを横断的に把握
- **設計書→手順書→実装の流れを一気通貫で指示**: 1つのチャットで企画から開発まで
- **記事執筆**: ログと体験を元に4,000字超の記事を自動生成。トーン・文体の指定も可能
- **作業ログの自動記録**: タイムラインやファイル一覧を構造化して記録
- **X投稿文の作成**: 文字数制限（140字）を守りつつ、ハッシュタグ付きの投稿文を複数パターン生成

### 記事になるポイント
- 「日本語1行のプロンプトで何ができるか」の具体例集
- Coworkの「チャットタブ↔コードタブ」の使い分け
- 「手順書を作らせて、その手順書を実行させる」という2段階指示テクニック

## 2. Claude in Chrome × Note投稿の自動化

### やれたこと
- **Note記事の自動投稿**: エディタにテキスト入力→ハッシュタグ設定→公開ボタンまで自動操作
- **ProseMirrorエディタへの入力方法**: `document.execCommand('insertText', false, text)` でJS経由挿入
- **ハッシュタグの手動削除・追加**: 自動生成されたタグ(#1,#2...)を削除し、正しいタグに差し替え
- **公開設定（無料/有料）の切り替え**

### やれなかったこと
- **画像アップロード**: ブラウザからローカルファイルにアクセスできないため断念
  - base64変換→JS注入を試みたが、サイズ制約で実用的でなかった
  - `upload_image` ツールはスクリーンショットIDが必要で、ファイルからは使えなかった
- **Noteの下書き一覧からの編集操作**

### 記事になるポイント
- 「AIにNote投稿を自動化させたら、画像だけ手動だった」という体験談
- ProseMirrorエディタとの格闘記（技術者向けネタ）
- 「2日連続投稿」の自動化でNoteの投稿習慣を作る方法

## 3. Claude in Chrome × X投稿の自動化

### やれたこと
- **投稿テキストの自動入力**: compose画面にテキスト入力→下書き保存
- **下書き保存のフロー自動化**: 閉じるボタン→保存ダイアログ→保存

### やれなかったこと
- **画像添付**: ファイルアクセスの制約で自動化できず
- **直接投稿**: 安全のため下書き保存までにとどめた（誤投稿防止）

### 記事になるポイント
- 「X投稿もAIに任せられるが、画像と最終投稿は人間がやるべき」
- 下書き→確認→投稿という安全なフロー

## 4. Claude Code CLI × スクショ自動化

### やれたこと
- **シミュレータスクショ一括撮影**: `xcrun simctl io booted screenshot` で9枚自動撮影
- **フォルダ作成→撮影→git commit まで一気通貫**
- **撮影のタイミング制御**: アプリ操作→待機→スクショの流れ

### やれなかったこと
- **デスクトップアプリのスクショ完全自動化**: `screencapture -w` はウィンドウクリックが必要
- **Cowork画面のスクショ**: Claude Code CLIからCowork自身を撮ることはできない（同じアプリ）

### 記事になるポイント
- 「スクショ撮影もAIに任せたら9枚一瞬だった」
- シミュレータは完全自動、デスクトップは半自動という温度差
